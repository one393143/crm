CRP — 客戶 360 度戰情系統
=========================================

一套專為業務拜訪設計的本地查詢工具。
輸入客戶名稱關鍵字，立即從本地資料庫彙整出聯絡人、訂購紀錄與所有歷史 POS 交易，
無需等待 API、無需開啟多個系統分頁。


---------------------------------------------
系統架構
---------------------------------------------

採用「全量同步 + 本地查詢」設計：

  【同步階段】手動執行 --sync
      Kintone App 71   (聯絡人,      ~1,060 筆)  →  本地 SQLite
      Kintone App 69   (客戶訂購單,    ~743 筆)  →  本地 SQLite
      Kintone App 310  (POS 歷史,  ~14,852 筆)  →  本地 SQLite
      ── 共約 35 次 API，耗時約 3~4 分鐘 ──

  【查詢階段】日常使用
      → 直接查本地資料庫，0 次 API，瞬間回應

資料庫檔案：crp_cache.db（SQLite，自動建立於同目錄）


---------------------------------------------
檔案說明
---------------------------------------------

  kintone_v2.py  — 主程式（同步 + 查詢 + 顯示）
  db.py          — SQLite 資料庫模組（建表、寫入、查詢）
  crp_cache.db   — 本地資料庫（自動產生，勿手動修改）


---------------------------------------------
安裝需求
---------------------------------------------

  Python 3.10 以上
  requests 套件

安裝：
  pip install requests

Python 內建 sqlite3，無需額外安裝。


---------------------------------------------
使用方式
---------------------------------------------

1. 初次使用，先同步所有資料（約 3~4 分鐘）：

     python kintone_v2.py --sync

2. 查詢客戶（瞬間，0 次 API）：

     python kintone_v2.py 長榮超音波
     python kintone_v2.py 長榮          ← 模糊搜尋，關鍵字即可

3. 只更新單一 App（例如 POS 有新資料時）：

     python kintone_v2.py --sync 310
     python kintone_v2.py --sync 71
     python kintone_v2.py --sync 69

4. 查看資料庫同步狀態：

     python kintone_v2.py --status

5. 無引數執行，互動輸入：

     python kintone_v2.py


---------------------------------------------
查詢輸出說明
---------------------------------------------

每次查詢會依序列出三個區塊：

  [聯絡人]    App 71，所有符合的聯絡人
              欄位：姓名、職稱、電話、地址、Email、廠商代號 等

  [訂購單]    App 69，所有符合的訂購單
              欄位：訂購單號、日期、客戶PO號、金額、負責業務 等
              若有訂購明細（產品、數量、交期）亦一併展開顯示

  [POS 交易]  App 310，所有符合的歷史 POS 交易（最舊到最新）
              欄位：PO Date、Shipping Date、Product Type、Product No.、
                    Qty.、Sales Total、Cost、Margin、Invoice No. 等

所有欄位皆完整列出，不做人工篩選。


---------------------------------------------
何時需要重新同步（--sync）
---------------------------------------------

  ✓ 首次使用
  ✓ Kintone 上有新增聯絡人或新成交 POS 資料
  ✓ 不確定本地資料是否最新時（可先 --status 確認上次同步時間）

不需要每次查詢前都同步。一次同步，後續查詢皆走本地。


---------------------------------------------
API Token 設定（若需更換 Token）
---------------------------------------------

Token 直接寫在 kintone_v2.py 的 APPS 字典中：

  APPS = {
      71:  {"token": "...", "name": "聯絡人"},
      69:  {"token": "...", "name": "客戶訂購單"},
      310: {"token": "...", "name": "POS 歷史"},
  }

各 App 的 Token 取得方式：
  Kintone → 目標 App → ⚙️ 設定 → 自訂/整合 → API 權杖
  → 產生 Token，勾選「查看記錄」權限 → 更新應用程式


---------------------------------------------
已知限制
---------------------------------------------

  - 訂購單（App 69）的客戶名稱格式可能與 POS（App 310）不同
    （例如 POS 使用「C-長榮超音波(股)」），查詢時用中間段關鍵字
    即可同時命中兩邊。

  - Kintone API 每次最多回傳 500 筆，且 offset 上限為 10,000。
    本系統採用「$id 游標分頁」繞過此限制，可同步任意數量記錄。
